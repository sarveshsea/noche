About Dibs
AI-native production bidding for agencies and production companies
Dibs replaces the AICP spreadsheet with a smart estimation engine. It understands crew rates, union rules, equipment costs, and usage rights — then builds accurate bids in minutes instead of days. Every line item is traceable, every revision is tracked, and every budget tells a clear story.
Features
New Bid Setup
4-step guided wizard: select production type and template, fill project details, set shoot parameters, usage rights, and union rules — then let AI pre-fill every line item.
[1] 311-399+ line items cause errors [2] 300+ rows create cognitive overload [3] Manual rate lookups
Dashboard
At-a-glance view of bids out, revenue won, average budget, and deadlines. Win rate trend over 6 months with industry comparison. Recent bids table with status tracking.
[4] System doesn't externalize complexity [5] 17-25% win rate, 4-6 bids per win [6] No tool covers end-to-end workflow
Bid Board
Kanban pipeline across Draft, Submitted, Revision, and Won stages. Drag bids between columns, see totals per stage, and track your full pipeline at a glance.
[7] Version chaos across email attachments [8] Triple bid comparison is manual
AI Assistant
Conversational budget intelligence. Ask for totals, biggest cost drivers, savings opportunities, market benchmarks, or bid validation — get answers with inline action buttons.
[9] Bid form is a hidden calculation graph [10] Users forced to be the computation engine [11] Estimation relies on producer experience
Active Bid Line Items
Full AICP-structured line item editor with 9 category tabs. Borderless inline editing, collapsible sections, running totals, and data-driven rendering from a single source of truth.
[12] 4-level hierarchy flattened into rows [13] Form is rules engine + calc graph + report [14] 311+ line items with formula breakage

AICP Form Deep Analysis

Structure, UX Problems, and Opportunity Areas — the AICP bid form analyzed as a system, not just a form.

Key Insight: The problem is NOT "the form is long." The real problem is the system does not externalize complexity. Users are forced to be the computation engine that the software should be — doing mental simulations, manual dependency tracking, and memory-based navigation.

What the AICP Form Actually Is

The AICP form is NOT just a form. It is simultaneously:

A Rules Engine

OT calculations (1.5x/2x thresholds), union rate cards (SAG, IATSE), markup rules (25% production fee, 3% insurance, 15% travel handling).

A Calculation Graph

A DAG of financial calculations. Changing shoot days cascades to crew, equipment, locations, catering, and travel — but dependencies are implicit.

A Reporting Tool

Output must be clean, professional, AICP-format compliant, and shareable as PDF for agency/client review.

4-Level Hierarchy

LevelTypeExample
L1: SectionsMacro bucketsA-X: Pre-Pro, Production, Post, Other, Fees (expanded from A-N)
L2: CategoriesWithin sectionsCrew, Talent, Equipment, Locations, Travel
L3: Line ItemsGranular rowsDP @ $3,500/day x 3 days + OT = $12,250
L4: CalculationsComputed totalsSubtotals, markups (10-25%), contingency, grand total

7 Core UX Problems

1. Fragmented Navigation

Data spread across tabs with no persistent context. Users constantly scroll, jump, and lose position across 300+ rows.

2. High Cognitive Load

Must hold budget totals, category breakdowns, and dependency chains (crew ↔ equipment ↔ days) simultaneously.

3. Poor Data Density

Hundreds of rows in flat tables. No collapsing, progressive disclosure, or smart summaries.

4. Weak Calculation Transparency

Totals update silently. No diff tracking, change logs, or dependency mapping. "Why did this number change?"

5. Inefficient Input Flow

Same roles reused across bids with no templates, autofill, or reusable components.

6. No Search or Filter

Can't find a specific line item across 300+ rows, compare categories side-by-side, or audit specific cost areas.

7. Versioning Gaps

Version history is manual file naming. Comparison is manual side-by-side. Comments are external (email, PDF).

Jobs To Be Done

Construct fast, Understand where money goes, Adjust and see impact, Defend costs to stakeholders, Iterate across versions.

Atomic Design Only. Every generated component must declare an atomic level (atom, molecule, organism, template). Enforced in specs and codegen. No exceptions.
»
MCP Tool Decision Tree. use_figma (Official MCP) for design-system-aware operations. figma_execute (Console MCP) for raw Plugin API. Check Code Connect BEFORE creating anything.
»
Self-Healing Loop. Mandatory CREATE → SCREENSHOT → ANALYZE → FIX → VERIFY cycle (max 3 rounds) for all canvas operations.
»
Code Connect First-Class. Every component spec has a codeConnect field mapping Figma node IDs to codebase paths. Checked before generation, established after creation.
»
Multi-Agent Native. Multiple Claude instances connect on ports 9223-9232. Each shows a color-coded box widget in Figma (yellow=working, green=done, red=error).
»
AgenticUI Aesthetic. Monospace terminal-paper aesthetic. Dark theme for system UI, warm paper theme for generated output. Gold accent (#9D833E).
»
Skills Architecture. 10 skill files in skills/ with freedom levels (maximum, high, read-only, reference). Agents load relevant skills before acting.
»
Auto-Spec Engine. memoire pull auto-creates ComponentSpecs from Figma components. Infers atomic level, shadcn base, and props.
»
Single-Command Pipeline. memoire go runs the full pipeline in one command: connect → pull → auto-spec → generate → preview.
»
Token-Aware Codegen. Generated components inject CSS variable references from pulled design tokens automatically.
»
Motion Video Skill. /motion-video superagent for Apple-grade product animation, portfolio videos, motion tokens, and Figma → After Effects pipeline.
